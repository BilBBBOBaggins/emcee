# Граница переносимости — что завязано на рантайм, а что переносится

Этот пакет спроектирован под **рантайм Claude Code** (CLI/расширение от Anthropic). Перенос
регламента на другой рантайм (Codex CLI, Cursor, свой harness) — **ручная работа по этой карте**,
а не переключатель конфигурации. Почему не свап-механизм — [ADR-009](../docs/adr/009-portability-boundary.md):
оси «универсальное / процесс / привязка-к-рантайму» не ортогональны, а профили композиции —
owned-долг, уже зарубленный [ADR-001](../docs/adr/001-scope-process-overlay.md)/[ADR-006](../docs/adr/006-regimen-upgrade.md).

Эта карта — страховка под живой рынок фронтир-моделей: лидерство меняется, и переезд (например
Claude → Codex) вероятен. Карта делает разовый форк **быстрым**, не требуя машинерии между
переездами.

## Модель vs рантайм — две разные миграции

«Переехать с Claude на Codex» — это **одновременно** смена модели и смена рантайма, с разной ценой:

- **Смена рантайма** (Claude Code → Codex CLI): меняется *плумбинг* — авточтение `CLAUDE.md`,
  каталог `.claude/`, хуки, slash-команды, plan mode, `/rewind`. У другого рантайма свои механизмы
  (`AGENTS.md`, свой конфиг). Это переписывается руками. **Основная цена переезда — здесь, и её не
  автоматизирует ни один конфиг.**
- **Смена модели** (Opus → GPT-x): меняются только *заплатки под тики модели* (см. `origin:
  harness:claude-code`, подкласс model-tic ниже). Их мало.

## Нотация происхождения `origin:`

Расширение ярлыка правила (`[ID · класс · гейт]`, см. [constitution.md](constitution.md)). Метка
живёт **в теле правила как проза**, координата опциональна. Три значения:

| `origin:` | Что значит | При переезде |
|---|---|---|
| `universal` | Приём, не зависящий от модели и рантайма (контракт-первый цикл, декомпозиция, разведка/производство, адверсивный метод, метод дебага) | **Оставь идею**, переформулируй под идиому нового рантайма |
| `process-convention` | Процессное «хочется так» (роли, дни, команды `R D T`, артефакты пайплайна) | Конвенцию оставь, **перепаяй механизм** (субагенты/слэши → эквиваленты рантайма) |
| `harness:claude-code` | Привязка к рантайму Claude Code | **Перепиши руками** под новый рантайм (или выкинь, если механизма нет) |

Подкласс `harness:claude-code/model-tic` — заплатка под тик конкретной модели (напр. «нет
pre-existing failure как оправдания» — тик Клода). При смене модели — **пересмотри** под новую: у
неё другие тики.

**Стоп-условие метки (несущее, [ADR-009](../docs/adr/009-portability-boundary.md)).** `origin:`
остаётся вне owned-долга, только пока: (1) в теле правила, не во frontmatter; (2) не управляет
композицией; (3) не парсится/не валидируется для решений тулингом (`new-project.py`, `selftest.py`,
`regimen-doctor.py`, `sync-roles.py`). Пассивный render в handbook и видимость в diff — допустимы.
Нарушил любое → метка стала синхронизируемым артефактом → откатить.

Разметка `origin:` — **ленивая и инкрементальная**: проставляется по ходу касания правила, не
поголовным прогоном (поголовная простановка ради полноты — та самая инверсия усилие↔ценность,
которую зарубила панель). Большинство правил `core/` — `universal`; ярлык нужен там, где
происхождение неочевидно.

## Карта harness-зависимостей (что переписывать при форке)

Индекс того, что завязано на рантайм Claude Code. Это TODO-лист будущего ручного переноса:

| Зависимость | Где | На что менять в другом рантайме |
|---|---|---|
| Авточтение `CLAUDE.md` в начале сессии | корневой `CLAUDE.md` | механизм автоконтекста рантайма (напр. `AGENTS.md` у Codex) |
| Каталог `.claude/` (agents/skills/commands/hooks) | `.claude/` | эквиваленты субагентов/команд/хуков рантайма, либо прозовый режим |
| Субагенты с tool-scoping | `.claude/agents/*` | если нет субагентов — роли как прозовые режимы (роль = промпт) |
| Slash-команды (`/role`, `/panel`, `/kickoff`) | `.claude/commands/*` | команды/макросы рантайма, либо ручной запуск по md |
| Хуки-гейты (`check-loc`, `check-no-todo`, precompact) | `.claude/hooks/*` + `settings.json` | механизм хуков рантайма, либо вынести в CI/pre-commit |
| Plan mode, `/rewind`, авто-компакт контекста | поведение Claude Code | аналоги рантайма или ручная дисциплина |
| Числовые команды `R D T` как UX | `CLAUDE.md` + `roles.json` | UX-конвенция рантайма (та же идея, другой ввод) |
| Авто-память (auto-memory) | `core/memory.md` | механизм памяти рантайма + `CLAUDE.md`-иерархия |

**Что НЕ в этой таблице — переносится как идея:** методы `core/` (spec-driven, debugging,
adversarial-panel, principles, code-quality) — это `universal`, их суть от рантайма не зависит.

**AGENTS.md — субстрат контента, не гарантий.** Авточтение у Codex/Cursor идёт через `AGENTS.md` —
формат зрелый (де-факто стандарт AAIF/Linux Foundation), но он несёт **контент/конвенции**, а НЕ
аппаратные гарантии. Read-only/docs-only роли и хуки даёт **per-runtime permission-profile** (`.codex/`
и аналоги), не общий markdown. Поэтому на AGENTS.md кладётся нейтральный контент ролей; принуждение —
в оверлей рантайма (`origin: harness:<name>`), и его сила фиксируется матрицей гарантий ниже.

## Матрица гарантий per-runtime (роль/правило × рантайм × сила)

Что именно даёт каждый рантайм: **аппаратно** (OS/харнесс физически не даёт нарушить), **проза**
(honor-инструкция в `developer_instructions`/AGENTS.md — деградация, обязательство под запись) или
**отсутствует**. Источник по Codex — эмпирический тест живого бинаря `codex 0.138.0`
([docs/evidence/g2-findings.md](../docs/evidence/g2-findings.md)); реализация —
[`overlays/codex/`](../overlays/codex/).

| Гарантия (правило роли) | Роли | Claude Code | Codex (0.138.0) |
|---|---|---|---|
| **read-only** (нет записи/запуска) | reviewer, auditor | аппаратно — `tools: Read,Grep,Glob` | **аппаратно** — `sandbox_mode="read-only"`, seatbelt блокирует запись (G2-verified) |
| **full write** (код/тесты/билд) | developer, qa-e2e, debugger, devops | аппаратно — +Edit,Write,Bash | **аппаратно** — `sandbox_mode="workspace-write"` |
| **docs-only** (Write, но не код) | ba, qa-uat, sa, architect | аппаратно — `tools` без Edit/Bash | **проза** — `workspace-write`+accountability. G2 RED (живой агент): стабильный путь структурно неспособен (cwd всегда писибелен — агент записал `src/`), `workspace_roots`=тумблеры, per-path только через нестабильный `[permissions]`-enum |
| **scratchpad-only** (пишет разбор, не код) | red-team, blue-team, arbiter | аппаратно — scoped tools | **проза** — `workspace-write`+accountability (та же причина, что docs-only) |
| **Bash только узкий codex-фактчек** (судит предъявленное; не выносит приговор моделью) | arbiter | `tools` включает Bash, но роль-промпт ограничивает его узким фактчеком спорного эмпирического пункта (`adversarial-panel.md` §39) — **проза-граница, не аппаратная** | **проза** — нет per-tool deny на Codex |
| **slash-dispatch** `/role /panel /kickoff` | все | аппаратно — `.claude/commands/*` | **проза** — печатная конвенция `R D T` (нет кастомного slash) |
| **хуки-гейты** (LOC/precompact/no-todo) | — | аппаратно — `settings.json` hooks (срабатывают во всех режимах, вкл. headless) | **проза/accountability** — KL-7 (живой `codex exec`, 6 сессий): из конфига в headless хуки НЕ срабатывают даже с `--dangerously-bypass-hook-trust`; рабочие хуки = plugin-манифест + интерактивный TUI-trust. Для жёсткого гейта на Codex — CI/pre-commit |
| **skills** (progressive disclosure) | — | аппаратно — `.claude/skills/` (поддерживает `paths:` glob-скоупинг) | **аппаратно** — `.codex/skills/`, тот же примитив; тело `SKILL.md` общее (`name`+`description`), но **`paths:` — Claude-only** (Codex читает только name+description, discovery по description) |
| **авточтение входа** (контекст сессии) | — | аппаратно — `CLAUDE.md` | **аппаратно** — `AGENTS.md` (несёт контент сам, per-harness нативный вход — ADR-012; НЕ указатель на CLAUDE.md, файла CLAUDE.md в codex-проекте нет) |
| **авто-память** | — | аппаратно — нативная иерархия | частично — Codex memories (opt-in), иначе AGENTS.md |

**Чтение матрицы:** где «проза» — обязательство не исчезает, оно держится на дисциплине агента под
запись (accountability), а не на железе. **Обе живые верификации закрыты** (живой `codex exec`):
**KL-7** (6 сессий) — хуки из конфига в headless НЕ срабатывают → клетка хуков = accountability (жёсткий
гейт на Codex → CI/pre-commit); **docs-only** — стабильный путь структурно неспособен (агент записал в
`src/` при `writable_roots=["docs"]`, т.к. cwd всегда писибелен) → клетка docs-only = accountability.
Ни одной «pending»-клетки: матрица полностью заземлена на эмпирике 0.138.0. Детали —
[g2-findings.md](../docs/evidence/g2-findings.md). Slash-dispatch и docs/scratchpad-тиры на Codex —
осознанно принятая деградация (ADR-010/011), не дефект.

**Оговорка про `Task` (ADR-014, эмпирически подтверждена).** «docs-only **аппаратно**» для architect — это
«сам кода не пишет»; но architect несёт `Task` и **может делегировать** Bash-замер read-only-потомку
(прогон субагента подтвердил: architect добыл точные git/test/LOC-метрики без своего Bash). Значит
«аппаратность» docs-only для Task-несущих ролей **мягче, чем для чистого read-only**: дисциплина в том,
что делегат — read-only-замерщик, а не write-способный потомок (иначе граница обходится). Для метрик это
осознанный рабочий путь (см. [../roles/architect.md](../roles/architect.md)); для гарантии «не пишет код»
держится на том, что делегату не дают Edit/Write-на-код.

## Дисциплина вперёд: ядро держать чистым от рантайма

Чтобы переносимый объём не таял, при добавлении **нового** правила:

- Правило `universal` (метод, не плумбинг) **не должно** прорастать в `.claude/`-измы, имена
  slash-команд, plan mode, формат хуков. Описывай приём, а не инструмент.
- Привязку к рантайму выноси в правило с `origin: harness:claude-code` и/или дополняй карту выше —
  не размазывай её внутри универсального метода.
- Если новый приём по сути универсален, но удобно опереть на фичу Claude Code — раздели:
  универсальная суть в методе, опора на фичу — отдельной пометкой «на этом рантайме — так».

Чем чище `core/` от рантайма, тем больше при переезде переносится «как есть», и тем дешевле форк.

## Когда карты недостаточно — параллельные рантаймы (РЕАЛИЗОВАНО)

Карта выше рассчитана на **миграцию** (разовый форк с рантайма на рантайм). Случай «**гонять два
рантайма одновременно**» (Claude *и* Codex параллельно, живьём) — это уже не форк, а «общее
upstream-ядро + тонкие оверлеи под рантайм» (git-уровень, не зарубленный манифест). Этот случай прошёл
новую панель и **построен**:

- Решение — [ADR-010](../docs/adr/010-multimodel-core-overlays.md) (ядро + git-оверлеи под рантайм) +
  [ADR-011](../docs/adr/011-process-layer-and-multimodel-build.md) (разблокировка постройки, форма
  C+mapping).
- Реализация — [`overlays/codex/`](../overlays/codex/) (codex-оверлей), `new-project.py --harness
  claude-code|codex` (статическое копирование git-дерева, **без** парсинга меток), `sync-roles.py`
  как N-рантайм-эмиттер, матрица гарантий выше, tripwire долга в `selftest.py`.
- Раскладка (`.claude/` остаётся в нативной позиции, `overlays/` — только под не-дефолтные рантаймы) и
  documented mapping — [`overlays/README.md`](../overlays/README.md).

**Стоп-условие меток держится:** оверлей выбирается `--harness`-флагом (статическое копирование), а НЕ
машинным чтением `origin:`. Вернуть парсинг меток тулингом = снова новая панель (см.
[ADR-009](../docs/adr/009-portability-boundary.md)).
