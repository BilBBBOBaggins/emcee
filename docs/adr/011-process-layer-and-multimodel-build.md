# ADR-011: Process-слой (RFC) сейчас + разблокировка мультимодельной постройки (ADR-010) досрочным снятием гейта G1

Date: 2026-06-28
Status: Accepted (постройка разблокирована досрочным снятием эмпирического гейта G1 из [ADR-010](010-multimodel-core-overlays.md) — сознательное проектное решение с явным принятием риска)

> Решение принято прогоном адверсивной панели (red-team → blue-team → arbiter), см. [core/adversarial-panel.md](../../core/adversarial-panel.md).

Прямое продолжение [ADR-010](010-multimodel-core-overlays.md) (мультимодельность под гейтами) и [ADR-009](009-portability-boundary.md) (граница переносимости, стоп-условие меток).

## Коротко

Гейт необходимости мультимодельности (G1 из ADR-010) снят **досрочно** сознательным проектным
решением — эмпирика ещё не накоплена, риск слепой зоны принят явно и вынесен в мониторинг — и
запрошена финальная целевая архитектура. Предложенная **v1** добавляла ось `owner:
package|project` и делала метки `origin:`/`owner:` машинно-читаемыми (апгрейдер ключуется на `owner:`,
генератор вайрит по `origin:`). Панель установила: **v1 — регресс относительно уже принятого
ADR-010**, который тот же месяц прошёл полную панель и сознательно выбрал git-деревья-оверлеи +
статическое копирование без парсинга меток.

Решение (v2):

1. **Выбросить машинерию v1** — ось `owner:`, парсинг меток тулингом, физический split контента. Это
   фатально-сломано независимо от секвенсинга (нарушает стоп-условие ADR-009 и границу by-file
   ADR-006).
2. **Строить сейчас, harness-agnostic** — process-слой из RFC (skills-роутеры, фазы-контракты,
   depth-тиры, verify-формат) + уточнение про AGENTS.md + гейты в две плоскости. Чистый выигрыш
   качества, второй рантайм не нужен.
3. **Разблокировать постройку мультимодельного оверлея** в форме ADR-010 (git-деревья, статический
   `--wiring`, метки = проза) — досрочное снятие G1 и YAGNI-секвенсинга арбитра, цена принята явно.

## Контекст

После [ADR-010](010-multimodel-core-overlays.md) (мультимодельность принята как дизайн, но постройка
заблокирована полевыми гейтами G1/G2) гейт G1 **снят досрочно** сознательным проектным решением
(эмпирика необходимости ещё не накоплена — риск принят явно), и запрошена финальная архитектура. Параллельно два внешних RFC (Factorial: guidance-layer + routing
skills; advanced AI workflow) дали процессные паттерны, часть которых пакет ещё не имел.

Предложенная архитектура-v1 свела это в одну раскладку, но ввела:
- ось `owner: package|project` как **машинный ключ** апгрейдера;
- `origin:`-слой, по которому **генератор вайрит** namespace;
- физический split контента package/project.

Панель (red-team → blue-team → arbiter, с привлечением второй модели codex) с высокой конвергенцией
двух независимых моделей установила, что это переступает уже вынесенные вердикты:
- **ADR-006**: граница package/project «не делится по строке, вычислима **по файлу** by-content»
  (`{{`-плейсхолдеры vs clean). Ось `owner:` либо дублирует этот сигнал, либо делит неделимое
  (заполненный stack-файл = mixed).
- **ADR-009** (стоп-условие меток, [portability.md:39-43](../../core/portability.md)): `origin:` вне
  owned-долга **только пока** не управляет композицией и не парсится тулингом. v1 нарушает оба пункта
  **по дизайну**.
- **ADR-010**: уже выбрал статическое копирование git-деревьев `safe_copy_tree`/`--wiring` **без
  парсящегося манифеста** — именно чтобы не рецидивировать owned-долг ADR-001/006.

Факты сверены напрямую: ADR-009/010 существуют; `new-project.py` уже имеет
`safe_copy_tree` + `--wiring` и `origin`/`owner` не парсит; стоп-условие подтверждено в portability.md.

## Решение

### A. Выброшено (фатально, не спасается митигацией)

- **Ось `owner: package|project`** — отменена. Граница package/project = существующий by-content
  сигнал ADR-006 (`{{` vs clean), по файлу.
- **Парсинг меток тулингом** — отменён. Генерация = `safe_copy_tree`/`--wiring` (код уже есть).
- **Физический split контента** — не делается. Контент общий и harness-нейтральный; оверлеи = только
  плумбинг.

**Стоп-условие (несущее):** вернуть машинное чтение меток `origin:`/любой новой оси = осознанный
пересмотр ADR-009 новой панелью с принятием owned-долга, не молчком.

### B. Строится сейчас — process-слой, harness-agnostic (добор RFC, ADR-010 не покрывал)

- **skills = роутеры**: шаблон `Purpose / When to use / When NOT to use / decision-tree / key
  constraints / standard procedure / validation` + **quality-bar** (добавлять только если частый ∨
  дорогой провал ∨ повторяющиеся ревью-комменты ∨ агент стабильно выбирает не тот путь; НЕ для
  one-off / vague / policy-prose). Контент/шаблон/quality-bar строятся сейчас; per-harness
  discovery-проводка — позже.
- **pipeline = фазы-контракты**: фаза требует входной артефакт предыдущей → STOP, не имитация прогресса.
- **depth-тиры Inline / Atomic / Full** вместо бинарного micro/full в [constitution.md](../../core/constitution.md).
- **verify-формат** `COMPLETENESS / CORRECTNESS / EVIDENCE / ⚠ warnings` — форма exit-отчёта и вывода reviewer/qa.
- **coworker-not-executor** — несущим принципом.
- **AGENTS.md = субстрат контента/конвенций**, НЕ аппаратных гарантий (их даёт per-runtime
  permission-profile). Зрелость формата подтверждена (AAIF/Linux Foundation).
- **Гейты в две плоскости**: определение — в реестре конституции (11 non-negotiable); энфорсмент =
  per-runtime profile/хук. Mechanical-гейт без энфорсера на рантайме **деградирует в accountability**
  (под запись), обязательство не исчезает.
- **Метки `origin:` — проза** (4 оси: `universal` / `process-convention` / `harness:<name>` /
  `model:<name>`), ленивая разметка по касанию, не парсятся.

### C. Строится сейчас — мультимодельный оверлей в форме ADR-010 (постройка разблокирована)

G1 (необходимость) и YAGNI-секвенсинг арбитра сняты досрочно — сознательное проектное решение, цена
принята явно. Форма — строго ADR-010, без машинерии
меток:
- ядро (`core/` + `roles/*`-контент + `stack/architecture/domain`), вычищенное от harness-измов;
- независимые git-деревья оверлеев; генератор копирует выбранное статически;
  - **Форма (уточнение):** claude-code-оверлей **остаётся `.claude/` в
    нативной позиции**, НЕ двигается в `overlays/claude-code/`. `overlays/` заводится **только под
    не-дефолтные рантаймы** (`overlays/codex/`: `.codex/*.toml` permission-profiles, hooks,
    AGENTS.md-проводка, skills). Эквивалентность `.claude/ ≡ overlays/claude-code/` фиксируется
    абзацем доки (documented mapping). **Перенос отклонён:** консистентность мнимая —
    генератор всё равно кладёт в `target/.claude/` (асимметрия source↔target врождённая); перенос
    добавил бы symlink-хрупкость (Windows `core.symlinks=false`), self-host-break и churn 16 файлов
    при нулевой выгоде сверх эстетики. Оставить `.claude/` на месте + documented mapping строго
    доминирует по усилие↔ценность;
- `roles.json` как единый источник истины → `sync-roles.py` до N runtime-эмиттеров;
- матрица гарантий per-runtime (роль × рантайм × аппаратная/прозовая/отсутствует);
- `regimen-doctor` рапортует состояние, не наличие.

**Порядок:** (1) откат машинерии v1 → (2) process-слой (раздел B) → (3) нейтрализация ядра (~1.5 дня)
→ (4) оверлеи + N-эмиттеры + бэкстоп счёта файлов в `selftest.py` → (5) проверка профиля прав и хуков
Codex → матрица гарантий + doctor.

## Последствия

**Плюсы:** process-слой — чистый выигрыш качества сразу, без зависимости от второго рантайма; панель
поймала регресс v1 до вложения инженерных недель; мультимодельность строится в форме, уже прошедшей
панель (не рецидивирует owned-долг манифеста).

**Принятые цены/риски (явно):**
- **Досрочное снятие гейта G1** снимает единственный инструмент обнаружения риска: две недели энтузиазма
  → один дефолтный рантайм → второй оверлей гниёт, ограничивая каждое изменение ядра. Риск принят явно
  и отслеживается (лог потребности в открытых вопросах ниже).
- **Долг N оверлеев материален** (`.claude/` уже 25 файлов; codex-оверлей = второй продукт). Бэкстоп —
  счёт файлов в `selftest.py` (порог неподъёмности при добавлении роли).
- **Slash-dispatch на Codex** деградирует в прозу (`R D T`) — матрица гарантий фиксирует, не прячет.
- **«Нейтральное ядро» частично-ложно навсегда**: process-convention (`R D T`, дни) на Codex
  остаётся печатной идиомой даже после нейтрализации ядра — матрица показывает честно.

**Открытые вопросы (узнаются полем/тестом):**
- [ ] Нейтрализация ядра — grep-sweep harness-измов по всему ядру (~1.5 дня). Предусловие выноса ядра.
- [ ] Проверка профиля прав на Codex — `docs-only.toml` + 3 атаки записи в `src/` (≤0.5 дня). Если запись
      проходит, docs-only на Codex понижается до прозы (правит матрицу гарантий, не хоронит).
- [ ] Тест активации хуков Codex — `Stop`/`PreCompact` реально срабатывают.
- [ ] Лог потребности в мультимодельности ведётся в ходе постройки (окно 6 недель). Не блокер (гейт
      снят), но даёт данные для будущего решения «свернуть ли codex-оверлей»: если потребность = только
      перекрёстная проверка, оверлей был не нужен (codex уже даёт вторую модель).

## Рассмотренные альтернативы

- **v1 как поставлен (ось `owner:` + парсинг меток + split).** Отклонено: регресс относительно
  ADR-010, нарушает стоп-условие ADR-009 и границу by-file ADR-006 (вердикт панели — фатально).
- **Отложить оверлей до зелёного G1 (рекомендация арбитра, YAGNI-секвенсинг).** Отклонено:
  цена постройки сейчас принята явно (сознательное проектное решение). Process-слой (B) от этого не зависит и шёл бы сейчас в любом случае.
- **Новая панель по пересмотру стоп-условия ADR-009** (легализовать машинное чтение меток). Не
  понадобилась: цели разделения достигаются by-content сигналом ADR-006 + статическим `--wiring`, без
  меток-машинерии.
- **Provider (вендор) как ось регламента.** Отклонено: на регламент не влияет; harness и model —
  достаточные оси.
