# ADR-010: Мультимодельность — ядро + git-оверлеи под рантайм, под двумя полевыми гейтами

Date: 2026-06-28
Status: Accepted — постройка разблокирована и выполнена через [ADR-011](011-process-layer-and-multimodel-build.md)
(G1-гейт снят досрочно сознательным проектным решением; G2 docs-only закрыт; codex-оверлей построен). Формулировка «заблокирован G1/G2 — постройка не начата» в теле ниже — **историческая**, на момент написания 2026-06-28.

> Решение принято прогоном адверсивной панели (red-team → blue-team → arbiter), см. [core/adversarial-panel.md](../../core/adversarial-panel.md).

Это **прямое продолжение [ADR-009](009-portability-boundary.md)**: там посылка была «один фактический рантайм» → свап-машинерия зарублена; здесь посылка изменилась (оператор намерен реально гонять 2+ рантайма), что ADR-009 сам пометил как триггер новой панели.

## Коротко

Оператор хочет **полноценную мультимодельность** — не быть запертым в одной модели/рантайме, а
реально работать на Claude Code и Codex параллельно (рынок фронтир-моделей живой, лидерство
меняется). Панель проверила архитектуру «общее **ядро** (метод + контент ролей) + **оверлеи**,
воспроизводящие слой принуждения под каждый рантайм».

Вердикт: **структура здорова, фатального дефекта нет — но строить пока нельзя.** Решение стоит на
двух фактах, которых на текущих данных НЕТ, и оба узнаются только полем, а не дебатом:

- **G1 — необходимость наблюдаема, а не желаема.** «2+ рантайма» сейчас — намерение, не наблюдение.
  Главный риск (red): две недели энтузиазма, потом один дефолтный рантайм, а второй оверлей **гниёт,
  ограничивая каждое изменение ядра** — ровно судьба зарубленной свап-машины. Гейт: 6-недельный
  лог необходимости с негеймабельным критерием.
- **G2 — docs-only на Codex действительно аппаратный.** Панель эмпирически доказала (живой тест
  бинаря codex), что permission-profile Codex даёт пер-путь read/write на уровне OS-песочницы →
  docs-only роли (BA/QA/SA пишут доки, но не код) воспроизводимы **в железе**, не деградацией. Но
  точная TOML-форма профиля не дожата — это 0.5-дневный тест.

Ключевое уточнение из r2: **если необходимость держится на «нужна вторая модель для перепроверки»
(cross-check), оператору нужен не оверлей, а вторая модель — которая у него уже есть через codex.**
Оверлеи оправданы, только если упор в **рантайм-специфичные примитивы/лимиты** (capability-block).

## Контекст

После [ADR-009](009-portability-boundary.md) (переносимость = ручной форк по карте, без машинерии)
оператор сформулировал более сильную цель: не разовая миграция, а **параллельная работа на
нескольких рантаймах**. ADR-009 заранее назвал это триггером новой панели.

Решение заземлено **фактической картой возможностей рантаймов** (`runtime-capability-map.md`,
собрана с codex + прямым фетчем доков): метод (`core/`) переносится прозой без потерь; слой
принуждения (`.claude/`) воспроизводим на Codex/Cursor частично. В ходе панели карта была
**уточнена эмпирически** — blue прямым тестом бинаря codex 0.138.0 показал, что docs-only тир на
Codex воспроизводим аппаратно (permission-profile, энфорс seatbelt/landlock), опровергнув
первоначальную оценку «дыра». Невоспроизводимых аппаратно гарантий осталось **одна** —
пользовательские slash-команды (`/role`, `/panel`), которые на Codex деградируют в печатную
конвенцию `R D T`.

## Решение

**Расщепить пакет на upstream model/runtime-agnostic ЯДРО + независимые git-дерево-ОВЕРЛЕИ
принуждения под рантайм. Охват v1 — Claude Code + Codex; Cursor отложен. Постройка — только после
зелёных G1 и G2.**

Структура (принята как дизайн; строится после гейтов):

- `core/` + `roles/*` (контент) + `stack/architecture/domain` — **общее ядро**, вычищенное от
  harness-измов (предусловие P3).
- `overlays/claude-code/` (= нынешний `.claude/`), `overlays/codex/` (`.codex/agents/*.toml` с
  permission-profiles, hooks, AGENTS.md-проводка, skills) — **независимые деревья**; генератор
  копирует выбранное статически (`safe_copy_tree`, класс `--wiring`), **не** ветвя тулинг по
  `--runtime` и **без** парсящегося манифеста (иначе рецидив owned-долга ADR-001/006).
- `roles.json` — уже SSOT; `sync-roles.py` расширяется с Claude-таргетов до **N runtime-эмиттеров**
  (P4). Skill-генерация, сегодня знающая Claude-схему (`new-project.py:471`), выносится в
  runtime-specific эмиттер/оверлей.
- **Матрица гарантий per-runtime** (обязательна): «роль × рантайм × гарантия (аппаратная / прозовая
  / отсутствует)». Честно показывает, где харнесс сильнее, где слабее. Slash-dispatch —
  единственная клетка «прозовая на Codex».
- `regimen-doctor` рапортует **состояние, не наличие** («hooks active/trusted», не «файл есть»).

### Полевые гейты — ДО любой постройки (несущие предусловия)

**G1 — наблюдаемая необходимость (окно 6 недель).** Лог append-only, одна
строка в день задачи **вперёд** (не задним числом): `date · task · runtime · why_tag ·
blocking_artifact`. `why_tag` — закрытый список. «Необходим» = только `capability-block` (упёрся в
отсутствующий примитив/жёсткий лимит, с артефактом) или `cross-check` (метод требовал второй
модели codex, с артефактом-разбором). Прочее = «удобен», не считается.
- **GREEN (строить):** ≥6 `capability-block` по ≥4 разным неделям, оба рантайма незаменимы ≥1 раз
  каждый. **`cross-check` для решения о постройке не засчитывается** (если порог держится на нём —
  нужна вторая модель (codex), не оверлей).
- **RED (откат к ADR-009):** ≤3, ИЛИ сгрудились в <4 недели, ИЛИ один рантайм ни разу не был
  незаменим. **AMBER (+3 нед однократно):** 4–5 при выполненных распределении и двусторонности.
- Негеймабелен: тег вперёд (не ретро-суждение); «необходим» требует артефакта, не мнения;
  распределение по неделям + двусторонность + исключение `cross-check` закрывают лазейки;
  незаписанная задача дефолтит в RED (лень → консервативный откат, не ложный green).

**G2 — docs-only на Codex аппаратный (≤0.5 дня).** Собрать `docs-only.toml`
(write только в `docs/`, deny в `src/`; точная форма `FilesystemPermissionToml` — 1 тест) + 3 атаки
записи в `src/` (`apply_patch`, raw-shell, MCP). GREEN: все три блокируются. RED: хоть одна проходит
→ Codex docs-only = honor-system, роли деградируют в прозу (матрица фиксирует, решение не хоронится).
Параллельно — тест активации хуков.

### Последовательность после зелёных G1+G2

1. Grep-sweep `R D T`/Claude-нарратива по **всему** будущему ядру (не три файла: также `roles/*`,
   `core/memory.md`, `core/principles.md`, `task-protocol.md`, `pipeline.md`, `stack/_TEMPLATE.md`,
   `architecture/agentic-workflows.md`), нейтрализовать или вынести в оверлей (~1.5 дня).
2. `overlays/codex/` + N-эмиттеры `sync-roles.py`, **обязать счётом файлов** в `selftest.py` (порог
   неподъёмности долга).
3. Матрица гарантий + doctor-адаптер.

## Последствия

**Плюсы:** честная мультимодельность с аппаратными гарантиями там, где рантайм их даёт (read-only
reviewer и docs-only — на обоих рантаймах при G2-green); owned-долг **впервые оправдан** (покупает
реальную параллельную работу, а не ноль, как в ADR-009); структура не рецидивирует ошибку
парсящегося манифеста (git-деревья, не манифест); матрица гарантий не прячет деградацию, а показывает её.

**Риски (последствия решения, не отложенные решения):**

- **Slash-dispatch на Codex — прозовая деградация**, не аппаратная. `R D T` остаётся печатной
  конвенцией; матрица гарантий это фиксирует. Принято осознанно.
- **Долг N оверлеев материален.** Бэкстоп — счёт файлов в `selftest.py`: если добавление роли
  трогает слишком много файлов × рантаймов, долг признан неподъёмным до постройки.
- **Активация хуков ≠ наличие**: doctor должен рапортовать состояние, иначе ложное чувство
  принуждения.

**Эмпирическая верификация (единственные настоящие TODO — узнаются полем, не дебатом):**

- [ ] G1-гейт: 6-недельный лог необходимости. Зелёный → строить; красный →
      откат к ADR-009, эта структура не реализуется. Это гейт-решение, не отложенный дизайн:
      дизайн принят, поле решает, активировать ли его.
- [ ] G2-гейт: рабочий `docs-only.toml` + 3 атаки записи (≤0.5 дня). Красный →
      docs-only на Codex понижается до прозы (правит матрицу, не хоронит).
- [ ] Тест активации хуков Codex (`Stop`/`PreCompact` реально срабатывают).

## Рассмотренные альтернативы

- **Строить сейчас, без полевых гейтов.** Отклонено: «2+ рантайма» сейчас — намерение, не
  наблюдение; преждевременная постройка рискует мёртвым оверлеем, гниющим и ограничивающим ядро
  (блокирующее возражение).
- **Хук-docs-enforcement на Codex** (PreToolUse вето записи вне `docs/`). Отклонено как **ложная
  гарантия**: shell-запись утекает мимо хука (Codex-доки: PreToolUse — не полная граница). Заменено
  permission-profile (аппаратный).
- **`--runtime`-ветвление генератора/doctor/selftest + парсящийся манифест.** Отклонено: рецидив
  owned-долга ([ADR-001](001-scope-process-overlay.md)/[ADR-006](006-regimen-upgrade.md)). Выбрано —
  копирование независимых git-деревьев без интерпретации.
- **Lowest-common-denominator** (выкинуть невоспроизводимые гарантии везде, включая Claude, ради
  единого харнесса). Отклонено: жертвует аппаратной гарантией у Claude ради симметрии — прямое
  нарушение north star «качество > простота».
- **Отдельные форки без общего ядра** (по форку на рантайм). Отклонено: дублирует метод, ядро
  рассинхронизируется; общее upstream-ядро + оверлеи дешевле в сопровождении (при условии G1).
- **Прозовый режим везде, без оверлеев.** Жизнеспособный fallback (метод переносится прозой без
  потерь), и это то, к чему откатывает красный G1. Не выбран как цель: на Codex теряются аппаратные
  read-only/docs-only гарантии и хуки, которые там **воспроизводимы** — харнесс с оверлеем
  существенно сильнее голой прозы (при условии, что 2+ рантайма реально нужны).
- **Cursor в v1.** Отклонено сейчас: docs-only там тоже бинарный, slash деградирует — добавлять
  отдельным инкрементом после Claude+Codex.
