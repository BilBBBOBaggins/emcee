## 1 Приглашение участника по email

**Статус:** 🟢 Production
**Файлы:** `components/features/InviteTeammateModal.tsx`, `components/features/InviteButton.tsx`, `lib/api/invites.ts`, `internal/transport/invite_handler.go`, `internal/service/invite_service.go`

### Описание

На странице команды пользователь нажимает «Invite teammate», вводит email и отправляет приглашение. Система создаёт pending-инвайт и ставит письмо в очередь. Пользователь видит подтверждение или понятную ошибку.

### Сценарий 1.1: Happy path — отправка приглашения

**Предусловие:** пользователь залогинен, открыта страница своей команды, у него есть право приглашать.

| Шаг | Действие пользователя | Ожидаемый результат | Slack делает так же? |
|-----|----------------------|---------------------|----------------------|
| 1 | Нажимает «Invite teammate» (`inviteButton`) | Открывается модалка (`inviteModal`), курсор в поле email (`inviteEmailInput`), кнопка Send (`inviteSubmit`) неактивна | Да |
| 2 | Вводит `newdev@example.com` | Кнопка Send становится активной | Да |
| 3 | Нажимает Send | Кнопка показывает спиннер и неактивна; через 1–2 сек — toast (`inviteToast`) «Invitation sent to newdev@example.com», модалка закрывается | Да (Slack показывает «Invite sent») |

### Сценарий 1.2: Edge case — невалидный / пустой email

**Предусловие:** модалка открыта.

| Шаг | Действие пользователя | Ожидаемый результат | Slack делает так же? |
|-----|----------------------|---------------------|----------------------|
| 1 | Оставляет поле пустым | Кнопка Send неактивна, отправить нельзя | Да |
| 2 | Вводит `not-an-email` | Кнопка Send остаётся неактивной (клиентская валидация) | Да |
| 3 | Исправляет на `ok@example.com` | Кнопка Send активируется | Да |

### Сценарий 1.3: Error path — участник уже приглашён

**Предусловие:** на `dup@example.com` уже есть активный pending-инвайт в этой команде.

| Шаг | Действие пользователя | Ожидаемый результат | Slack делает так же? |
|-----|----------------------|---------------------|----------------------|
| 1 | Вводит `dup@example.com`, нажимает Send | Модалка остаётся открытой; под полем инлайн-ошибка (`inviteError`) «This person already has a pending invite» | Да (Slack: «already invited») |
| 2 | Меняет на новый email | Ошибка исчезает, можно отправить | Да |

### Тестовые данные

- Новый инвайт: `newdev@example.com`
- Дубль: `dup@example.com` (требует заранее созданного pending-инвайта)
- Невалидный: `not-an-email`
